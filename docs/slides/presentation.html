01 // the problem

RAG shouldn't require 12 SaaS subscriptions

  • Most RAG tools lock you into cloud vector DBs with per-query pricing
  • Your documents leave your network for embedding and inference
  • No single tool gives you search, agents, graphs, eval, and sync
  • Existing CLIs are proof-of-concept toys, not production systems

ragcli runs 100% local. Your data never leaves your machine.

02 // overview

What ragcli actually is

A production-ready RAG system built on Oracle Database 26ai for vector search and Ollama for local LLM inference. Ships with multi-agent reasoning, knowledge graphs, hybrid search, feedback loops, session memory, an evaluation suite, and live document sync.

# one line install
pip install oracle-ragcli

08 // agents

Multi-agent Chain of Thought

Planner

Decomposes the query into subtasks and builds an execution plan

Researcher

Retrieves relevant chunks from the vector store and knowledge graph

Reasoner

Analyzes retrieved context, identifies connections, resolves conflicts

Synthesizer

Produces the final answer with citations and confidence scores

Full trace stored in Oracle DB. Inspect any step via CLI, API, or TUI.

09 // knowledge graph

LLM-powered knowledge graph

  • Entity/relationship extraction from every chunk
  • Max 10 entities per chunk (configurable)
  • Deduplication at 0.9 similarity threshold
  • Multi-hop traversal (default: 2 hops)
  • Stored in Oracle graph tables
# config.yaml
knowledge_graph:
  enabled: true
  max_entities: 10
  dedup_threshold: 0.9
  max_hops: 2

13 // sync

Live document sync

3 sync modes

  • Watchdog: filesystem monitoring
  • Git polling: track repo changes
  • URL polling: fetch remote docs
# Watch a directory
$ ragcli sync add /docs \
  --pattern "*.md"

# Watch a git repo
$ ragcli sync add ~/project \
  --type git

Diff-chunking: only re-embeds changed sections, not the entire document.

22 // schema

17 Oracle tables

Core

  • DOCUMENTS: metadata, format, size
  • CHUNKS: text + VECTOR column
  • QUERIES: user queries + results
  • SESSIONS: multi-turn conversations

Extended

  • KG_ENTITIES / KG_RELATIONS
  • AGENT_TRACES: CoT pipeline logs
  • FEEDBACK: user ratings
  • EVAL_RUNS / EVAL_RESULTS
  • SYNC_SOURCES / SYNC_EVENTS

HNSW index on embeddings. JSON metadata columns. All managed by "ragcli db init".
